How to use flybrid

1. A <table> with a <tbody>

A flybrid table is just a simple <table> element with the .flybrid class added to it, a width value of 100%, and a <tbody> element added inside to wrap the table's rows and columns (don't forget!). There's no need for addional <div> elements, conditional code, and no complex CSS rules to write.

2. Use <th> for cells

Each cell in a flybrid table must be a <th> element, instead of the usual <td>. This is to make the stock email client on Android 4.4 (KitKat) devices respect overrides of the display CSS attribute.

3. 'Double-declare' widths for each cell

Use pixel widths values, and declare the width using the width attribute on each <th> AND apply the same width in CSS using the style attribute.

The width attribute value is for the less capable email clients, and will work while the table structure is still intact. The style value will be used when the table structure breaks down, and the cells become inline-block elements. It also helps columns resize properly in Outlook's 120 DPI mode

4. Avoid padding at <th> level

You can expect strange results if you try to add padding the <th> cells of a flybrid table. If you want to add padding to your columns, you should use a nested table (see examples below) or add spacing some other way.

5. Eliminate all gaps between <th> & <tr> elements

A bug introduced in Apple's Mail app in iOS9 means that you need to eliminate any space between the opening and closing <th> & <tr> elements within your flybrid table. If not, the Mail app will add additional spacing between cells, force them to stack, even when the viewport is wide enough to display the desktop layout.
